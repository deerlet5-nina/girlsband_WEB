# Girls Band Animation 网站开发文档

## 总体设计

### 网站框架设计
网站首页（index.html）展示了本网站的主题，围绕女子乐队动画主题，基于文字和图片，提供了对网页内容的描述，包括对四部动画作品的总体介绍，以及各乐队成员的角色展示，更加直观；同时提供动画主题曲试听功能，供浏览者欣赏。轻音少女页（k-on/index.html）展示了轻音少女的乐队故事与角色介绍，图文结合进行介绍。孤独摇滚页（bocchi/index.html）展示了孤独摇滚的乐队历程与音乐特色，并提供链接指向官方音乐资源，以供浏览者进一步了解。MYGO!!页（mygo/index.html）展示了MYGO!!的乐队发展与音乐风格，并提供音乐试听模块。Girls Band Cry页（gbc/index.html）展示了Girls Band Cry的乐队特色与音乐作品，介绍了他们的代表曲目与创作理念。乐队搭配游戏页（bandgame/index.html）提供了互动游戏功能，让用户体验组建乐队的乐趣，同时提供音乐创作工具。关于我们页（about.html）提供了与本网站相关的信息，包括建立本网站的初衷等，同时提供友情链接功能。最后，登录页（login.html）提供用户登录功能，登录后，为浏览者提供个性化内容推荐功能。

### 布局设计
本网站所有页面均遵循统一的总体布局设计，分为四个主要区块：

1. Header区：
   - 网站Logo：展示网站名称和主题
   - 主题切换按钮：支持明暗主题切换
   - 时钟显示：实时显示当前时间
   - 用户登录入口：提供登录/注册功能
   - 背景音乐控制：可控制背景音乐播放

2. Menu区：
   - 主导航菜单：包含首页、四部动画专题页、乐队游戏等链接
   - 子导航菜单：各专题页面的二级导航
   - 快速导航：提供页面内快速跳转
   - 搜索功能：支持内容搜索

3. Content区：
   - 轮播展示：展示精选内容和最新动态
   - 主要内容：根据页面类型展示不同内容
     * 首页：动画作品概览和特色内容
     * 专题页：角色介绍、音乐资源、相关视频
     * 游戏页：乐队搭配游戏界面
   - 侧边栏：显示相关推荐和最新动态
   - 音乐播放器：支持在线试听功能

4. Footer区：
   - 版权信息：网站版权声明
   - 快速链接：主要页面的快速访问
   - 关于我们：网站信息链接
   - 联系方式：提供反馈渠道
   - 友情链接：相关网站链接

### 布局方法
整体布局采用CSS（page.css）和div混合布局，各区域具体布局方法如下：

1. Header区布局：
   - 使用flex布局实现水平排列
   - Logo区采用相对定位，实现悬停放大效果
   - 搜索框使用固定宽度和圆角设计
   - 时钟和登录区采用flex布局自适应排列

2. Menu区布局：
   - 主导航使用flex布局实现水平菜单
   - 子导航采用grid布局实现多列展示
   - 快速导航使用固定定位，实现悬浮效果
   - 搜索框采用弹性布局自适应宽度

3. Content区布局：
   - 轮播图使用绝对定位实现全屏效果
   - 作品展示区采用grid布局实现响应式卡片
   - 音乐播放器使用固定定位，实现悬浮效果
   - 侧边栏采用flex布局实现垂直排列

4. Footer区布局：
   - 使用grid布局实现多列内容排列
   - 版权信息采用flex布局居中显示
   - 快速链接使用flex布局实现水平排列
   - 联系方式采用grid布局实现图标和文字对齐

     * 返回：首页、其他专题页

     * 可跳转至：角色详情、音乐资源、相关视频
     * 返回：首页、其他专题页

## 2. 开发平台

### 2.1 开发环境
我们使用以下工具进行开发：

- 操作系统：Windows 10/11、macOS 10.15+、Linux (Ubuntu 20.04+)
- 
- 版本控制：Git 2.40.0+、GitHub Desktop 3.2.0+

### 2.2 技术栈
网站采用以下技术构建：

前端：
- HTML5 + CSS3 + JavaScript (ES6+)
- jQuery 3.7.1 用于DOM操作
- Font Awesome 6.4.0 提供图标支持
- Google Fonts 提供中文字体支持

后端：
- PHP 8.1.0+ 作为主要开发语言
- MySQL 8.0.0+ 用于数据存储
- Apache 2.4.0+ 或 Nginx 1.20.0+ 作为Web服务器
- PDO 用于数据库操作

### 2.3 运行环境
服务器需要满足以下要求：

- PHP 8.1.0 或更高版本
- MySQL 8.0.0 或更高版本
- Apache 2.4.0+ 或 Nginx 1.20.0+
- 必要的PHP扩展：PDO、PDO_MySQL、mbstring、json、session、openssl
- 支持现代浏览器：Chrome 90+、Firefox 90+、Safari 14+、Edge 90+

### 2.4 本地开发
推荐使用以下集成环境进行本地开发：

- XAMPP 8.1.0+（Windows）
- WAMP 3.2.0+（Windows）
- MAMP 6.0+（macOS）
- phpMyAdmin 5.2.0+ 用于数据库管理

## 8. 登录页面功能描述

### 8.1 登录功能

登录页面提供用户身份验证功能，用户需要输入学号、密码以及验证码才能完成登录。登录表单的登录按钮初始状态为失效，用户无法直接提交表单。

系统首先对验证码进行校验，当验证码输入框失去焦点时，立即触发验证码校验流程。如果用户输入的验证码正确，系统会弹出"验证成功：验证码正确，再次点击按钮登录"的提示信息，同时激活登录按钮，允许用户进行后续操作；如果验证码输入错误，登录按钮将保持失效状态，系统会弹出"验证失败：验证码错误！"的提示信息，用户需要重新输入正确的验证码。

当用户点击登录按钮提交表单后，系统会对学号和密码进行验证。验证规则要求学号必须以"233"开头，且必须是10位数字；密码必须为"123456"。如果验证通过，系统会显示登录成功的提示信息，并在2秒后自动跳转到网站首页；如果验证失败，系统会显示相应的错误提示信息，用户需要检查并重新输入正确的信息。

整个登录过程中，系统会实时显示验证状态和错误提示，帮助用户快速定位和修正输入错误。登录成功后，系统会保存用户的登录状态和学号信息到本地存储中，以便后续使用。
