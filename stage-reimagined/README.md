# Girls Band Stage Reimagined 新版网站说明

这是在**不改动旧版源文件**前提下，重新生成在 `stage-reimagined` 目录中的新版站点。

它不是对旧版首页做微调，而是一次并行重构：

- 首页不再堆满全部内容，而是变成“总入口 / 导览大厅”
- 四支乐队分别拥有独立主页面
- 乐队搭配小游戏被完整恢复，并重新适配美术风格
- 颜色、材质、按钮和动线都围绕四部作品的气质重新设计

## 设计目标

这一版的重点不是“更亮”，而是“更有区分度、更像专题站、更接近作品气质”。

设计方向包括：

- 首页只负责分流和推荐，不再承担所有内容
- `K-ON!`、`孤独摇滚`、`MyGO!!!!!`、`Girls Band Cry` 各自拥有不同的主视觉语汇
- 标题区大量使用边角柔和的几何形状
- 功能按钮不再只是普通链接，而是成为视觉与交互的一部分
- 播放、切换、推荐、小游戏等功能在新版视觉系统中统一表达

## 页面结构

### 1. 新版首页

- 入口文件：`stage-reimagined/index.html`

首页承担的是“演出场馆大厅”的角色，主要负责：

- 展示当前主舞台
- 在四支乐队之间切换入口
- 提供今夜推荐与快捷体验
- 跳转到小游戏
- 记录并继续用户上次浏览的舞台

### 2. 四个独立乐队主页面

- `stage-reimagined/bands/k-on.html`
- `stage-reimagined/bands/bocchi.html`
- `stage-reimagined/bands/mygo.html`
- `stage-reimagined/bands/gbc.html`

每个页面都不是简单换文案，而是有自己的：

- 独立专题结构
- 视觉主题与色彩系统
- 场景切换模块
- 主打歌曲与内容卡片
- 成员展示与氛围信息区
- 专属按钮语言和版式节奏

### 3. 搭配小游戏

- `stage-reimagined/bandgame.html`

小游戏已经完整接回新版站点，保留并增强了：

- `15` 元预算机制
- 借钱按钮
- 波奇连续借钱效果
- 同团或特定组合加成
- 详细演出评价
- `CRYCHiC` 相关隐藏事件
- 音效开关

## 新版主要功能

### 全局音乐播放器

新版统一使用全局播放器，不再依赖每个子页各写一套播放器逻辑。

支持：

- 点歌后立即进入播放队列
- 上一首 / 下一首
- 播放 / 暂停
- 进度条拖动
- 加载中、Ready、Now Playing、Paused 等状态反馈

相关文件：

- `stage-reimagined/js/app.js`

### 多页面主题切换

首页和乐队页共享一套数据驱动逻辑，可根据乐队切换：

- 主色与辅助色
- 封面图
- 标题文案
- 推荐内容
- 舞台标签

相关文件：

- `stage-reimagined/js/bands-data.js`
- `stage-reimagined/js/app.js`

### 更鲜明的四团风格区分

这一版不是单纯“换颜色”，而是按作品气质拉开差异：

- `轻音少女`：暖木、茶会、放学后社团空气
- `孤独摇滚`：低照度 Live House、霓虹和贴纸噪点
- `MyGO!!!!!`：雨夜、雾蓝、留白和剧场感
- `Girls Band Cry`：街灯、柏油、锈红与都市反抗感

相关文件：

- `stage-reimagined/css/site.css`
- `stage-reimagined/js/bands-data.js`

### 互动增强

新版补充了更偏专题站的互动体验，包括：

- 随机进入某支乐队
- 继续上次浏览的舞台
- 首页导航高亮
- 滚动显现动画
- 设计感更强的主操作按钮

## 运行方式

### 重要说明

新版页面会引用仓库根目录下的公共资源，例如：

- `images/`
- `music/`

所以**不要只单独把 `stage-reimagined` 文件夹作为站点根目录启动**。推荐直接在整个仓库根目录启动本地服务。

### 推荐启动方式

在仓库根目录执行：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000/stage-reimagined/index.html
```

## 与旧版的关系

新版是并行重构版本，和旧版的关系是：

- 旧版源码保留不动
- 新版在单独目录中重建页面与交互
- 公共图片和音乐资源继续复用仓库根目录下的原资源
- 便于在不破坏旧站的前提下继续大胆迭代

## 适合的使用场景

- 作为当前主推的展示版本
- 用于继续做视觉升级和交互扩展
- 作为“多子页面专题站”的后续开发基础
- 用于对比旧版和新版在信息架构上的差异
